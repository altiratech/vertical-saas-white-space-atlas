# Vertical SaaS White-Space Atlas Implementation Entry Brief

Purpose: keep the first engineering passes aligned with the founding industry-selection loop.

## Current Product Center

Build the smallest serious loop:

1. ingest CBP, SBA, and BLS
2. normalize one inspectable industry table
3. compute wedge scores with visible components
4. publish a static ranked explorer

## First Build Blocks

### 1. Data model foundation

Lock schemas for:
- industry cells
- source artifacts
- score inputs
- score outputs
- evidence

### 2. Core scripts

Initial repo work should support:
- source fetch and cache
- industry normalization
- scoring contract enforcement
- site data build

### 3. Web surfaces

Initial web surfaces should support:
- ranked table
- filter panel
- detail pane
- rationale and caveat view

## Explicit Defers

Do not treat these as founding blockers:
- state-by-state rollout
- map-first experience
- workflow or company drill-down
- user accounts

## Design Guardrail

This product should feel like a serious decision surface, not a generic startup trend board.
