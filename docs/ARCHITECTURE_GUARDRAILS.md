# Vertical SaaS White-Space Atlas Architecture Guardrails

## Scope Guardrails

- Build the ranked industry loop first.
- Keep the first user and workflow fixed.
- Do not broaden into a market-intelligence suite before the ranking is clearly useful.

## Data Guardrails

- Core objects:
  - `industry_cell`
  - `source_artifact`
  - `score_input`
  - `score_output`
  - `evidence`
- Keep raw, normalized, and inferred layers separate.
- Synthetic data is prohibited.
- No score without visible evidence or caveats.

## Product Guardrails

- ranked move first
- evidence before flourish
- table-first before fancy overview
- one clear recommendation per row

## Repo Guardrails

- Keep the repo standalone.
- Prefer scripts and static output before backend expansion.
- Use explicit schemas before clever abstractions.
